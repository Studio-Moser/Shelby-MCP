# Development Guide

## Prerequisites

- Go 1.22+
- SQLite3 (included via `modernc.org/sqlite` — no system dependency)

## Building

```bash
# Build the binary
go build -o shelbymcp ./cmd/shelbymcp

# Build for all platforms
GOOS=darwin GOARCH=arm64 go build -o shelbymcp-darwin-arm64 ./cmd/shelbymcp
GOOS=darwin GOARCH=amd64 go build -o shelbymcp-darwin-amd64 ./cmd/shelbymcp
GOOS=linux GOARCH=amd64 go build -o shelbymcp-linux-amd64 ./cmd/shelbymcp
GOOS=linux GOARCH=arm64 go build -o shelbymcp-linux-arm64 ./cmd/shelbymcp
GOOS=windows GOARCH=amd64 go build -o shelbymcp-windows-amd64.exe ./cmd/shelbymcp
```

## Testing

```bash
# Run all tests
go test ./...

# Run with verbose output
go test -v ./...

# Run specific package
go test ./internal/db/...

# Run with race detector
go test -race ./...
```

## Project Structure

```
cmd/shelbymcp/    # Entry point
internal/
  db/             # SQLite operations (thoughts, edges, FTS, vectors, migrations)
  mcp/            # MCP protocol server (stdio JSON-RPC)
  tools/          # MCP tool implementations
  config/         # CLI flags and configuration
skills/           # Shipped scheduled skills
docs/             # Documentation
```

## Key Design Decisions

### Pure Go SQLite (modernc.org/sqlite)

We use the pure Go SQLite implementation (no CGO) so the binary is truly zero-dependency and cross-compiles without a C toolchain. This means:

- `go build` works on any platform without installing SQLite
- The binary is self-contained
- No dynamic linking issues
- FTS5 is included

### WAL Mode

The database opens in WAL (Write-Ahead Logging) mode for concurrent read access during writes. This is important when the Forage skill and the MCP server access the same database simultaneously.

### UUID Primary Keys

Thought and edge IDs are UUIDs (v4), generated by the server. This avoids collisions when thoughts are captured from multiple AI tools simultaneously, and makes future cross-database merging possible.

### JSON Metadata Fields

`topics`, `people`, and `metadata` are stored as JSON text in SQLite. This keeps the schema simple while allowing flexible metadata. Agents provide this data at capture time.

## Adding a New MCP Tool

1. Create a handler in `internal/tools/`
2. Register the tool definition in `internal/mcp/tools.go`
3. Add routing in the tool router
4. Write tests
5. Update README.md tool table
6. Update CHANGELOG.md

## Adding a New Forage Task

1. Update `skills/shelby-forage/SKILL.md` with the new task description
2. The task should use existing MCP tools (list, search, update, link, capture)
3. Document the task in README.md's Forage section
4. Update CHANGELOG.md
